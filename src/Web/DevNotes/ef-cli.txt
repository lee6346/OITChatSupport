
$ dotnet ef migrations add <FileName>
	- make sure csproj ItemGroup does not have "condition=Target.Framework...."
	dotnet ef migrations add <FileName> -o Data\Migrations

$ dotnet ef migrations remove
	- removes (goes back) one migration command point

$ dotnet ef database update
	- implement the migration by makings changes to the database
$ dotnet ef database drop
	- undo the most recent db implementation of the migration
